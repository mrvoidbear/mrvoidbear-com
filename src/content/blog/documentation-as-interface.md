---
title: "Documentation as interface"
description: "Treat docs like product: discoverable, honest, and tested like code."
pubDate: 2026-04-01
---

If your documentation is hard to find, it does not exist. If it lies, it is worse than nothing—because trust compounds, and so does distrust.

Good docs behave like a good API: predictable structure, obvious entry points, and errors that tell you what to do next. That means indexing (search, sidebar, cross-links), ownership (who updates this when the system changes?), and a willingness to delete pages that no longer reflect reality.

The cheapest win is to write for the next on-call engineer at 2am: short titles, concrete examples, and links to dashboards or runbooks. The second win is to test what you can—smoke-test snippets, validate example requests, and add CI checks when docs live next to code.

Documentation is not “extra.” It is the interface between your system and every human who touches it.
